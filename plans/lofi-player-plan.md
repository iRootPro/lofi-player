# lofi-player — план проекта

> Компактный, красивый TUI-плеер для лоу-фай и эмбиент-стримов.
> Для прослушивания во время сосредоточенной работы.
> Написан на Go с Bubble Tea.

Этот документ — единственный источник истины по проекту. Описывает
*что* строим, *зачем* и в каком *порядке*. **Решения, помеченные как
«зафиксировано», не пересматриваются без явного согласия пользователя.**
Остальное — рекомендации, действуй по ситуации.

---

## 1. Обзор

`lofi-player` — терминальный музыкальный плеер для фоновой музыки во
время сосредоточенной работы. Пользователь слушает стримы лоу-фай /
chillhop / эмбиента (сейчас — в YouTube) и хочет быстрый,
клавиатурный, визуально полированный TUI вместо вкладки в браузере.

Идентичность продукта стоит на трёх вещах:

1. **Реальный, удобный плеер** — стримы должны играть стабильно
2. **Инструмент для фокуса** — pomodoro и сессионная статистика
   first-class, а не пристройка
3. **Объект ремесла** — палитра Tokyo Night, vim-ключи, полировка
   уровня «такое скриншотят и шарят в TUI-комьюнити»

### 1.1 Цели

- Воспроизведение интернет-радио стримов (lofi, chillhop, ambient,
  focus) надёжно
- Быстрый, ненавязчивый клавиатурный UX
- Красивый дефолтный визуал (Tokyo Night), сменные темы
- Pomodoro и сессионная статистика — встроенные фичи
- Жизнь в терминале — работает в tmux, дружит с i3/sway/yabai

### 1.2 Не-цели

- **Управление музыкальной библиотекой** — это не Spotify и не
  каталогизатор
- **Социальные фичи** — никаких шарингов, друзей, комментариев
- **Мобильные платформы, GUI, веб** — только терминал, только Linux + macOS
- **Эквалайзеры, DSP, аудио-эффекты** — базовой громкости достаточно
- **Замена mpv/cmus/ncmpcpp** — другая ниша; намеренный фокус на стримах
  + работе

### 4.2 Ключевые принципы

**Иммутабельный Update в Bubble Tea.** Все методы `Update` на `Model`
используют value-receivers. Изменения происходят на локальной копии
`m`, потом возвращается `m, cmd`. Pointer-receivers породят гонки и
тонкие баги — никогда не использовать на главном Model.

**Аудио живёт в своей горутине.** Пакет `audio.Player` владеет
соединением с mpv. Никогда напрямую не трогает TUI. Коммуникация
однонаправленная: плеер эмитит события (метаданные, ошибка, EOF),
которые доставляются в Bubble Tea как кастомные `tea.Msg` через
долгоживущий subscription `tea.Cmd`. TUI отправляет команды (play,
pause, volume), вызывая методы плеера изнутри фабрик `tea.Cmd` —
никогда инлайном в `Update`.

**Theme → Styles.** `Theme` — это плоская цветовая палитра. `Styles` —
мешок предсобранных `lipgloss.Style`, которые использует рендерер.
Смена темы в рантайме — это просто `m.styles = NewStyles(theme.Lookup(name))`.

**Один Model, композированные подмодели.** По мере роста приложения
вынеси подвьюхи (список станций, панель помодоро) в свои маленькие
модели со своими Init/Update/View. Корневой Model встраивает их и
диспатчит события. Не давай корневому Model превратиться в god-object.

**Кастомные сообщения, не каналы в Update.** Способ интеграции горутин
в Bubble Tea: долгая работа возвращает `tea.Cmd`, чья функция возвращает
`tea.Msg`. Читаешь из канала? Оборачиваешь это в Cmd, возвращающий
Msg, потом переподписываешься, возвращая ту же Cmd из следующего
Update. Никогда не блокируй внутри `Update`.

### 4.3 Аудио-архитектура подробно (референс для Phase 1)

```
┌──────────────────────────┐
│      Bubble Tea TUI      │
│   (главная горутина)     │
└──────────┬───────────────┘
           │ вызовы методов (Play, Pause, SetVolume)
           │   изнутри фабрик tea.Cmd
           ▼
┌──────────────────────────┐
│    audio.Player          │
│  (владеет mpv-соединением)│
└──────────┬───────────────┘
           │ JSON IPC через Unix-сокет
           ▼
┌──────────────────────────┐
│       mpv subprocess     │
│ --idle --input-ipc-server=/tmp/lofi-player-XXXX.sock
└──────────────────────────┘

События обратно идут через Go-канал внутри Player, в TUI приходят через
subscription tea.Cmd, который перевзводится после каждого приёма.
```

Player выставляет наружу:

```go
func NewPlayer(ctx context.Context, opts Options) (*Player, error)
func (p *Player) Play(url string) error
func (p *Player) Pause() error
func (p *Player) Resume() error
func (p *Player) SetVolume(percent int) error
func (p *Player) Events() <-chan Event   // закрывается при Stop
func (p *Player) Close() error
```

Типы `Event` (запечатанные через интерфейс): `MetadataChanged{Title,
Artist}`, `PlaybackStarted{}`, `PlaybackPaused{}`, `Error{Err}`, `EOF{}`.

TUI-слой оборачивает каждый `Event` в соответствующий `tea.Msg` и
обрабатывает в `Update`.

Темы — чистые Go-структуры (без внешних файлов в MVP). Встроенные:

- **Критерии готовности** — как понять, что сделано

### Phase 0 — каркас UI

**Цель.** Bubble Tea-приложение, которое рендерит layout, ходит по
списку станций и «фейкает» play/pause и громкость. Аудио ещё нет.

**Что нужно сделать.**
- Bootstrap проекта (`go.mod`, структура пакетов из §4.1, без `audio/`,
  `pomodoro/`, `state/`)
- `internal/config/config.go` — загрузка YAML из XDG-пути, запись
  дефолтов на первый запуск
- `internal/theme/theme.go` — структура `Theme`, `TokyoNight()`, `Lookup()`
- `internal/tui/` — `Model`, `Update`, `View`, `KeyMap`, `Styles`
- `README.md` с инструкциями по запуску

**Дефолтный конфиг** идёт со следующими надёжными стримами:

```yaml
theme: tokyo-night
volume: 60
stations:
  - name: SomaFM Groove Salad
    url: https://ice1.somafm.com/groovesalad-256-mp3
  - name: SomaFM Drone Zone
    url: https://ice1.somafm.com/dronezone-256-mp3
  - name: SomaFM Deep Space One
    url: https://ice1.somafm.com/deepspaceone-128-mp3
  - name: Radio Paradise Mellow
    url: https://stream.radioparadise.com/mellow-128
```

**Биндинги.**
- `j`/`↓`, `k`/`↑` — навигация по станциям
- `space` — play/pause выделенной станции
- `+`/`=`, `-`/`_` — громкость вверх/вниз (шаг 5%)
- `?` — переключить help (компактный ↔ полный)
- `q`, `ctrl+c` — выход

**Критерии готовности.**
- `go run .` запускает TUI, layout соответствует §5.1
- Курсор движется по станциям через `j`/`k`
- `space` тоглит фейковый флаг `playing`, текст статуса меняется
  ("live" / "paused"); играющая станция помечена `♪` в списке
- Бар громкости отражает стейт, ограничен 0–100
- `?` тоглит help; `q` выходит чисто (alt-screen восстанавливается)
- Конфиг записан в `~/.config/lofi-player/config.yaml` на первом
  запуске, парсится на следующих

**Подводные камни.**
- Bubble Tea-приложения должны использовать `tea.WithAltScreen()` для
  полноэкранных режимов
- `tea.WindowSizeMsg` — первое сообщение после `Init`; гарди
  рендеринг до момента, когда `m.width != 0`, иначе на старте мигнёт
  пустым широким фреймом
- `key.Binding` для пробела — это литерал `" "`, не `"space"`
- Не импортируй ничего из `audio/` — этого пакета пока нет

### Phase 1 — Аудио MVP

**Цель.** Реальное воспроизведение. Выбираешь станцию, жмёшь space,
музыка играет.

**Что нужно сделать.**
- `internal/audio/player.go` — публичный тип `Player`, API из §4.3
- `internal/audio/mpv.go` — IPC-клиент. Спавнит mpv с
  `--idle=yes --no-video --no-terminal --input-ipc-server=<socket>`,
  использует Unix-сокет, шлёт команды как JSON-строки, читает события
  в отдельной горутине
- `internal/audio/icy.go` — парсит изменения свойства `metadata` от
  mpv в `Title` / `Artist` (эвристический split по `" - "`; fallback —
  вся строка как title)
- `internal/tui/messages.go`, `commands.go` — мост между
  аудио-событиями и Bubble Tea
- Обновить `Model`: добавить `*audio.Player`, реальный `playingURL`,
  `currentTrack`
- `Update` обрабатывает: `MetadataChangedMsg`, `PlaybackStartedMsg`,
  `PlaybackErrorMsg`, `EOFMsg`
- Subscription `tea.Cmd` (`waitForEvent`), который читает из
  `player.Events()` и возвращает соответствующий Msg, потом Update
  возвращает ту же Cmd обратно для перевзвода

**Критерии готовности.**
- mpv обязателен в рантайме; если не на `$PATH`, приложение выходит с
  понятным сообщением до запуска TUI
- Нажатие `space` на станции запускает воспроизведение в течение ~2с
- `space` снова — пауза; ещё `space` — продолжение (без перезапуска
  стрима с нуля, где это возможно)
- Выбор другой станции и `space` переключает стрим чисто (без наложения)
- Метаданные трека появляются в панели now-playing в течение нескольких
  секунд после смены трека (server-side ICY-update)
- Изменения громкости слышны и сохраняются между переключениями станций
- Ошибки воспроизведения (DNS, 404, разрыв сети) показываются как
  transient toast в области help-бара, музыка не крашит приложение
- Выход из приложения чисто останавливает mpv (нет orphan-процессов)

**Подводные камни.**
- IPC-сокет mpv нужно прибирать на закрытии; используй временную
  директорию (`os.MkdirTemp("", "lofi-player-*")`) и `os.RemoveAll` в shutdown
- mpv эмитит *очень много* событий; подписывайся выборочно через
  `observe_property` на `pause`, `volume`, `metadata`, `media-title`,
  `idle-active`. Не парси весь firehose
- Протокол mpv IPC — JSON по строкам; один JSON-объект на строку.
  Используй `bufio.Scanner` со щедрым буфером (дефолт 64КБ;
  ICY-метаданные на некоторых стримах больше — подними до 1МБ)
- В каждой команде `request_id`, чтобы матчить ответы; держи маленький
  in-memory map
- ICY-метаданные стримов приходят как изменение свойства `metadata` с
  ключом `icy-title` (иногда `title`). Не предполагай конкретные имена —
  логируй неизвестные ключи на debug и инспектируй
- Не блокируй главный TUI-цикл вызовом `player.Play()` — оборачивай в
  `tea.Cmd`
- macOS: mpv из Homebrew работает из коробки. Никаких специальных
  аудио-разрешений для стримов

### Phase 2 — UX-полировка

**Цель.** Приложение ощущается законченным. Подходит как ежедневник.

**Что нужно сделать.**
- Добавить темы: `catppuccin-mocha`, `gruvbox-dark`, `rose-pine`
- Клавиша `t` циклит темы; выбор сохраняется
- `internal/state/state.go` — загрузка/сохранение state.json в
  `$XDG_STATE_HOME/lofi-player/state.json`: последняя станция,
  громкость, тема, статистика помодоро (наполнится в Phase 3)
- Spring-анимация `harmonica` на бар громкости
- Mini mode (`m` тоглит)
- Флаг `--statusline` — печатает одну строку статуса и выходит (для tmux)
- Toast-компонент: переходящие сообщения в нижней части экрана
  (ошибки, "добавлено в избранное" и т.д.), автодисмисс через 3с
- Полный help-экран как центрированная карточка с группированными
  биндингами

**Критерии готовности.**
- Перезапуск приложения: возвращает на последнюю станцию, последнюю
  громкость, последнюю тему
- `t` плавно циклит темы, все видимые элементы обновляют цвета
- `m` тоглит mini mode, не теряя воспроизведение
- `lofi-player --statusline` печатает одну цветную строку и выходит с 0
- Ошибки больше не требуют копаться в логе

**Подводные камни.**
- Spring-анимациям нужна tick-команда (`tea.Tick`); убедись, что тики
  прекращаются по достижении steady state, иначе 60fps idle CPU
- Persistence — best-effort: ошибка записи стейта никогда не крашит
  приложение, только лог

### Phase 3 — Pomodoro

**Цель.** First-class focus tool. То, ради чего это приложение
существует относительно `cmus`.

**Что нужно сделать.**
- `internal/pomodoro/pomodoro.go` — чистая машина состояний (без I/O),
  состояния: `idle`, `focus`, `short_break`, `long_break`. Длительности
  конфигурируемы. Счётчик раундов (по дефолту: long break после каждой
  4-й focus-сессии)
- TUI-интеграция: панель помодоро появляется в правой колонке когда
  активна
- Клавиша `p` стартует/останавливает помодоро-сессию
- На переходе `focus → break`: автопауза музыки (конфигурируемо)
- На переходе `break → focus`: автовозобновление музыки (конфигурируемо)
- Опционально: переключение на отдельный «брейковый» список станций во
  время брейков (тишина или только эмбиент)
- Сессионная статистика: время прослушанного сегодня, текущий streak
  (число подряд идущих дней с хотя бы одной focus-сессией)
- Уведомления на переходах:
  - macOS: `osascript -e 'display notification ...'`
  - Linux: `notify-send` если есть
  - Fallback в TUI: заметный toast

**Дополнения в конфиг.**
```yaml
pomodoro:
  focus_minutes: 25
  short_break_minutes: 5
  long_break_minutes: 15
  rounds_until_long_break: 4
  auto_pause_on_break: true
  auto_resume_on_focus: true
  break_stations: []  # пусто = тишина во время брейков
```

**Критерии готовности.**
- `p` стартует focus-сессию; таймер обратного отсчёта виден
- На нуле переходит в брейк, поведение музыки следует конфигу
- Сегодняшний счётчик "listened" растёт только во время focus-сессий
- Streak переживает рестарты (читается из state.json)

**Подводные камни.**
- Не дрейфуй: храни *абсолютное время окончания*, а не оставшуюся
  длительность, которую декрементируешь. Пересчитывай оставшееся на
  каждом тике через разницу. Иначе sleep/suspend ноута сдвинет таймер
- Брейковый список станций независим от основного — отдельный
  YAML-ключ, отдельная обработка курсора

## 7. Соглашения о коде

### Go-стиль

- Стандартные `gofmt` / `go vet` / `goimports`. CI падает на дифах
- Оборачивание ошибок: всегда `fmt.Errorf("doing X: %w", err)` —
  читателю нужна цепочка
- Никакого глобального стейта. Передавай `*Player`, `*Config` и т.д.
  явно
- Doc-комментарии на каждый экспортированный символ; полные
  предложения, начинающиеся с имени символа
  (`// Player controls mpv via JSON IPC.`)
- Тесты живут рядом с тестируемым файлом как `*_test.go`. Используй
  table-driven тесты с подтестами, когда кейсов больше одного
- Никакого `init()` — только конструкторы

### Логирование

- `charmbracelet/log` настроен на запись в файл
  `$XDG_STATE_HOME/lofi-player/lofi-player.log` (ротация по размеру,
  ~5МБ)
- **Никогда не логировать в stdout/stderr — ломает TUI**
- Уровни: `Debug` для IPC-трафика, `Info` для смен стейта, `Warn` для
  recoverable проблем, `Error` для того, что должен видеть пользователь
- Флаг `--debug` поднимает уровень до Debug

### Соглашения Bubble Tea

- Один файл — одна забота: `model.go`, `update.go`, `view.go`,
  `keys.go`, `styles.go`, `messages.go`, `commands.go`. Не объединять
  `update` и `view` даже если короткие
- Сообщения в `messages.go`, имена через `XxxMsg` (`TickMsg`,
  `PlaybackErrorMsg`)
- Команды в `commands.go`, имена через `xxxCmd` (возвращают `tea.Cmd`)
- Не лезь в `audio.Player` из `View()`. View читает только из `m`.
  Все вызовы плеера идут изнутри фабрик `tea.Cmd`

### Нейминг

- Имена пакетов: lowercase, единственное число (`config`, не `configs`;
  `audio`, не `audios`)
- Аббревиатуры: капитализируй как слова (`URL`, не `Url`; `ICY`, не `Icy`)
- Имена файлов: snake_case если несколько слов (`now_playing.go`), но
  предпочитай короткие однословные

## 8. Стратегия тестирования

В этом проекте: **тестируй ядро, не chrome.**

### Что стоит покрывать

- `internal/config/`: round-trip YAML, дефолты, отсутствующие поля,
  поломанный ввод
- `internal/theme/`: `Lookup` возвращает правильную тему, корректно
  фолбэкает
- `internal/audio/icy.go`: парсинг ICY — table-driven тесты на странные
  реальные входы
- `internal/audio/mpv.go`: фрейминг IPC (корреляция request_id,
  частичные чтения, переполнения буфера scanner) — мокай сокет
- `internal/pomodoro/`: переходы машины состояний (чисто, легко
  тестируется)

### Что не стоит покрывать

- Рендеринг View (визуальную регрессию проще ловить глазами)
- Bubble Tea Update-логику для тривиальных keypress → state changes
- Всё, что требует реального mpv-процесса или сети — это интеграционные
  заботы, гонять руками во время acceptance фазы

### Интеграционное тестирование

Каждая фаза имеет ручные критерии готовности (§6). Это и есть
«интеграционные тесты». Прогоняй чек-лист перед закрытием фазы.

Для Phase 1 заведи документ test-streams с разными URL стримов,
упражняющими разные форматы метаданных, кодеки и стабильность.

## 9. Подводные камни

Не исчерпывающий список вещей, которые укусят, если не подготовиться.

### mpv IPC

- **Очистка сокета.** mpv оставляет файл сокета на ФС если убит резко.
  Всегда создавай сокет в tempdir и `os.RemoveAll` на shutdown
- **Размер буфера.** `bufio.Scanner` дефолтит на 64КБ макс.
  ICY-метаданные на некоторых стримах больше. Подними буфер до 1МБ
- **Жизненный цикл процесса.** Трекай mpv-овский `*exec.Cmd`. На
  shutdown TUI шли `quit` через IPC, потом `cmd.Wait()` с таймаутом
  2с, потом `Kill()`
- **Pause vs idle.** `pause` и `idle-active` — разное. Используй
  `pause` для пользовательской паузы, `idle-active` для «ничего не
  загружено»

### Bubble Tea

- **Не вызывай `tea.Println` посреди работы.** Используй `tea.Printf`
  только вне alt-screen, иначе TUI получит мусорные артефакты. Для
  runtime-ошибок маршрутизируй в toast
- **Подписки нужно перевзводить.** Чтение из канала через `tea.Cmd`
  потребляет одно сообщение. Чтобы продолжать слушать, Update который
  обработал сообщение должен снова вернуть ту же Cmd. Забудешь —
  получишь ровно одно событие и тишину
- **Размер окна.** Первый `tea.WindowSizeMsg` приходит асинхронно
  после `Init()`. Не рендери layout-зависимые компоненты до получения
- **Медленные Cmd блокируют порядок.** Не клади долгую работу в
  функцию Cmd — выноси в горутину, которая вернёт сообщение позже

### YAML

- `yaml.v3` строгий с типами по умолчанию. Если пользователь поставит
  `volume: "60"` (строка) — парсинг упадёт. Либо кастомный
  анмаршаллер, принимающий оба, либо чёткая документация типов
- Пустые массивы vs nil — `stations: []` парсится в `[]Station{}`,
  `stations:` (без значения) парсится в nil. Считай оба случая «нет
  станций»

### XDG-пути

- Пользователи macOS всё равно получают `~/.config/lofi-player/` от
  `xdg.ConfigFile` — библиотека использует XDG-конвенции и на macOS,
  не Apple-овский `~/Library/Application Support`. Это намеренно для
  CLI-приложения и соответствует ожиданиям пользователей терминальных
  тулзов. Задокументируй путь в README

### Стримы

- Некоторые Icecast-серверы шлют chunked transfer, который mpv
  нормально жуёт, но какой-нибудь промежуточный прокси — нет. Если
  стрим «работает в браузере, не в mpv» — обычно дело в
  `Icy-MetaData: 1` в request-хедерах. Стоит дебаг-лога когда стримы
  падают

## 11. Открытые вопросы

Эти не блокируют работу, но нужно вернуться на соответствующей фазе:

- **Phase 2:** Стоит ли темам быть YAML-файлами
  (`~/.config/lofi-player/themes/*.yaml`) для пользовательских тем?

  другую станцию
- **Phase 4c:** Где живут эмбиент loop-файлы? Зашиты в бинарник
  (~10МБ лишних) или скачиваются на первое использование? У обоих свои
